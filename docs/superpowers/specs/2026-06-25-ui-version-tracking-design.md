# UI Version Tracking — Design Spec

## Status

- Approved: 2026-06-25
- Branch: `feat/ui-version-tracking`
- Target: PR to `main-with-dashboard`

---

## Overview

Add a version tracking system to the TradingAgents UI:

- Single source of truth: `VERSION` file at repo root
- Version displayed in header and Settings → About section
- CI enforces version bump when non-documentation/non-config files change

---

## 1. Version Source of Truth

**File:** `VERSION` (repo root)
**Starting value:** `1.1.0`
**Format:** Plain text, single line, no newlines

### Build-time Injection

Vite plugin reads `VERSION` at build time and generates `web/frontend/src/version.ts`:

```ts
export const VERSION = "1.1.0";
```

### Runtime Endpoint

Backend exposes `GET /api/version` returning `{"version": "1.1.0"}` so the Settings panel can show the version without rebuilding.

---

## 2. Version Display

### 2a. Header Badge (App.tsx)

In the header bar, next to the "TradingAgents" title, add a small pill badge:

```tsx
<span className="ml-2 px-1.5 py-0.5 text-[10px] font-mono font-medium text-slate-400 bg-slate-800/60 border border-slate-700/50 rounded">
  v{VERSION}
</span>
```

The `VERSION` constant is imported from `../version`.

### 2b. Settings Panel — About Section (SettingsPanel.tsx)

Add a new collapsible "About" section at the **bottom** of the Settings panel body (before the footer). It shows:

- Current version (from API endpoint `/api/version`)
- Note: "Version source: VERSION file at repo root"
- Toggle open/closed state

## 3. Backend Changes (app.py)

Add new endpoint:

```python
@app.get("/api/version")
def get_version():
    version_file = Path(__file__).resolve().parents[2] / "VERSION"
    version = version_file.read_text().strip() if version_file.exists() else "unknown"
    return {"version": version}
```

---

## 4. Vite Plugin (vite.config.ts)

Create a Vite plugin that:

1. Reads `VERSION` from repo root
2. Writes `web/frontend/src/version.ts` before build

```ts
function versionPlugin() {
  return {
    name: 'version-inject',
    buildStart() {
      const version = fs.readFileSync(path.resolve(__dirname, '../VERSION'), 'utf-8').trim()
      const outPath = path.resolve(__dirname, 'src/version.ts')
      fs.writeFileSync(outPath, `export const VERSION = ${JSON.stringify(version)};\n`)
    }
  }
}
```

Add `versionPlugin()` to the plugins array in `vite.config.ts`.

---

## 5. CI Version Check Job

**Trigger:** Push to `main`, `main-with-dashboard`, and PRs targeting them.

**Logic:**

1. Fetch base commit (origin/main or origin/main-with-dashboard) and head commit
2. Compare VERSION file content between base and head
3. If VERSION changed → PASS
4. If VERSION unchanged:
   - Get list of files changed (excluding blocklist)
   - If any non-blocklisted file changed → **FAIL** with message
   - If only blocklisted files changed → PASS

**Blocklist (version bump NOT required):**

```
*.md
docs/
LICENSE
.gitignore
.github/
*.yml
*.yaml
*.lock
*.log
.env*
```

**CI Failure Message (example):**

```
Version check failed: VERSION file was not updated.
Changes detected in: web/frontend/src/App.tsx
Update VERSION file before merging.
```

**Implementation approach:** GitHub Actions step using `git diff --name-only` and shell logic. No external action needed.
